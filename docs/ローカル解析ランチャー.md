# ローカル解析ランチャー手順

GitHub Pagesで開いたWebアプリから、PC上のYouTube解析サーバーを起動するための手順です。

## できること

- Webアプリが `http://127.0.0.1:8787/health` を見て、解析サーバーの起動状態を確認します。
- 未起動の場合、Webアプリの「解析サーバーを起動」ボタンから `mefx://start` を開きます。
- `mefx://start` は、このPCに登録した `scripts/mefx-local-analyzer.ps1` を起動します。
- Webページが開いている間はheartbeatを送ります。
- Webページを閉じてheartbeatが止まると、約1分後に解析サーバーが自動終了します。

## 初回だけ行うこと

PowerShellでプロジェクトのフォルダを開き、次を実行します。

```powershell
.\scripts\register-mefx-protocol.ps1
```

これで `mefx://start` という起動リンクがこのPCに登録されます。管理者権限は不要です。

## 解析ツールのインストール

初回起動時に、ランチャーが必要なツールを確認します。不足している場合は次の内容をインストールします。

- `server/node_modules`
- `ffmpeg-static`
- `yt-dlp`
- `demucs`
- `torchcodec`

手動で先に入れる場合は、次を実行します。

```powershell
.\scripts\install-analysis-tools.ps1
```

## 使い方

1. GitHub PagesのWebアプリを開きます。
2. 「YouTube音色コピー」ページを開きます。
3. 「ローカル解析サーバー」が「未起動」の場合、「解析サーバーを起動」を押します。
4. ブラウザが「外部アプリを開く」確認を出したら許可します。
5. 数秒後に「再確認」を押し、「起動中」になれば準備完了です。
6. YouTube URLと対象パートを選び、「解析してプリセット作成」を押します。

## 自動終了について

`scripts/mefx-local-analyzer.ps1` から起動した場合だけ、自動終了が有効になります。

- Webページが開いている間: 10秒ごとにheartbeatを送信
- Webページを閉じた後: heartbeatが止まる
- 約1分間heartbeatが来ない: 解析サーバーが終了

通常の開発用コマンド `.\scripts\run-server.ps1` で起動した場合は、自動終了しません。

## 登録を解除する

`mefx://start` の登録を外したい場合は、次を実行します。

```powershell
.\scripts\register-mefx-protocol.ps1 -Unregister
```

## 注意

- WebページがPC内のアプリを完全自動で起動することは、ブラウザの安全制限によりできません。
- 初回は必ずユーザー操作とブラウザの許可が必要です。
- YouTube解析はPC上で一時ファイルを作成しますが、解析後に削除されます。
