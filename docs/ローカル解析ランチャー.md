# ローカル解析サーバー手順

GitHub Pagesで開いたWebアプリから、PC上のYouTube解析サーバーを使うための手順です。

## 安全方針

Windowsセキュリティに引っかかりやすいため、Webページから `mefx://` でPowerShellを自動起動する方式は標準では使いません。

代わりに、PC側で自分で解析サーバーを起動します。Webアプリは起動状態を確認し、起動している時だけYouTube解析を送ります。

## できること

- Webアプリが `http://127.0.0.1:8787/health` を見て、解析サーバーの起動状態を確認します。
- 起動中はWebアプリからheartbeatを送ります。
- Webページを閉じてheartbeatが止まると、約1分後に解析サーバーが自動終了します。
- ローカル解析サーバーは、許可したWebアプリのOriginからのアクセスだけを受け付けます。

## 初回だけ行うこと

解析ツールを入れていない場合は、PowerShellでプロジェクトのフォルダを開き、次を実行します。

```powershell
.\scripts\install-analysis-tools.ps1
```

インストールされる主な内容:

- `server/node_modules`
- `ffmpeg-static`
- `yt-dlp`
- `demucs`
- `torchcodec`

## 使い方

1. PC側で次を実行します。

```powershell
.\scripts\start-local-analyzer.cmd
```

2. GitHub PagesのWebアプリを開きます。
3. 「YouTube音色コピー」ページを開きます。
4. 「ローカル解析サーバー」が「起動中」になっているか確認します。
5. 未起動表示のままなら「再確認」を押します。
6. YouTube URLと対象パートを選び、「解析してプリセット作成」を押します。

## 自動終了について

`scripts/start-local-analyzer.cmd` から起動した場合、自動終了が有効になります。

- Webページが開いている間: 10秒ごとにheartbeatを送信
- Webページを閉じた後: heartbeatが止まる
- 約1分間heartbeatが来ない: 解析サーバーが終了

通常の開発用コマンド `.\scripts\run-server.ps1` で起動した場合は、自動終了しません。

## Windowsセキュリティに引っかかった場合

Windowsセキュリティの「保護の履歴」を開き、検出された項目の場所を確認してください。

確認する場所:

- Windowsセキュリティ
- ウイルスと脅威の防止
- 保護の履歴

検出場所がこのプロジェクト内の `scripts/register-mefx-protocol.ps1` や `mefx://` 登録に関係するものだった場合、古い自動起動方式の影響です。現在の安全版では使いません。

古い `mefx://` 登録を解除するには、PowerShellで次を実行します。

```powershell
Remove-Item -Path "HKCU:\Software\Classes\mefx" -Recurse -Force -ErrorAction SilentlyContinue
```

検出場所がこのプロジェクト外、または知らないファイルの場合は、許可せずに内容を確認してください。

## 注意

- WebページがPC内のアプリを完全自動で起動することは、ブラウザの安全制限によりできません。
- 知らないWebサイトから外部アプリ起動の確認が出た場合は許可しないでください。
- 解析ツールの不足分は、無言ではインストールしません。
- YouTube解析はPC上で一時ファイルを作成しますが、解析後に削除されます。
- 許可Originを変えたい場合は、解析サーバー起動前に環境変数 `MEFX_ALLOWED_ORIGINS` にカンマ区切りで指定します。
