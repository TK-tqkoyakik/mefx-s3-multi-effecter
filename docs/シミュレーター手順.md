# ESP32なしで確認する方法

現段階ではESP32が接続されていなくても、Webアプリ内のシミュレーターで操作の流れを確認できます。

## 目的

シミュレーターで確認できること:

- Webアプリが起動する
- Manual Editorでプリセットを作れる
- YouTube Tone Matchで仮プリセットを作れる
- ESP32へ送ったつもりのプリセットがDevice Presetsに保存される
- AudioTelemetry風の入力/出力/マイク値が動く

確認できないこと:

- 実際のBluetooth通信
- 実際のESP32保存
- 実際の音声処理
- 実際の回路のノイズや音質

## 起動手順

### 1. 解析サーバーを起動

PowerShellを開いて実行します。

```powershell
.\scripts\run-server.ps1
```

### 2. Webアプリを起動

別のPowerShellを開いて実行します。

```powershell
.\scripts\run-web.ps1
```

### 3. ブラウザで開く

表示されたURLを開きます。

```text
http://localhost:5173
```

または最初からシミュレーターを有効にする場合:

```text
http://localhost:5173/?sim=1
```

### 4. Simulatorボタンを押す

画面右上の `Simulator` を押します。

成功すると:

```text
Simulator connected. ESP32なしで動作確認できます。
```

と表示されます。

## Manual Editorの確認

1. `Manual Editor` を開く
2. 名前を変更する
3. Guitar/Bassを切り替える
4. GainやDriveなどのスライダーを動かす
5. `Save` を押す
6. `Send to MEFX-S3` を押す
7. Device Presetsに追加されることを確認する

## YouTube Tone Matchの確認

1. `YouTube Tone Match` を開く
2. YouTube URLを入力する
3. Guitar/Bassを選ぶ
4. `Analyze and Create Preset` を押す
5. Manual Editorに推定プリセットが反映される
6. `Send to MEFX-S3` を押す
7. Device Presetsに追加される

この段階のYouTube解析は仮実装です。URL文字列から安定した値を作っているだけで、本物の音源分離はまだ行っていません。
