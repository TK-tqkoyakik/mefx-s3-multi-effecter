# 作業手順書

この手順書は、マルチエフェクターを完成に近づけるための作業順です。

## 全体の流れ

```text
1. PC環境を確認する
2. ESP32なしでWebアプリを動かす
3. シミュレーターでプリセット送信を試す
4. ESP-IDFとKiCadを入れる
5. ESP32単体でファームウェアをビルドする
6. 回路図と配線を完成させる
7. 部品を買う
8. 電源だけ確認する
9. 音声回路を確認する
10. Webアプリ + ESP32 + 音声回路をつなぐ
```

## 配線だけ見ながら作る入口

まず開くもの:

```text
docs/hardware/配線図を見ながら作る.html
```

詳しい表で確認するもの:

```text
docs/hardware/実体配線図.md
docs/hardware/配線表.csv
```

KiCadを開く場合:

```powershell
.\scripts\open-kicad.ps1
```

## Step 1: PC環境を確認する

PowerShellでプロジェクトフォルダを開き、以下を実行します。

```powershell
.\scripts\check-env.ps1
```

確認するもの:

| 項目 | 目的 | 現在の状態 |
| --- | --- | --- |
| Git | ファイル管理 | インストール済み |
| Node.js/npm | Webアプリと解析サーバー | `tools/` にダウンロード済み |
| ESP-IDF | ESP32ファームウェア作成 | まだ必要 |
| KiCad | 回路図/配線図作成 | インストール済み |

## Step 2: 解析サーバーを起動する

YouTube音色コピーで実際に音声を一時取得し、Demucsでパート分離して解析するには、先に解析ツールを入れます。

```powershell
.\scripts\install-analysis-tools.ps1
```

初回だけ、Demucsのモデルファイルを準備します。

```powershell
.\scripts\warmup-demucs.ps1
```

初回のモデル取得は数分かかることがあります。ベースはDemucsの `bass` ステム、ギターは `other` ステムを解析します。

現在はDemucsのAIパート分離が動作確認済みです。

- ベース: Demucsの `bass` ステムを解析
- ギター: Demucsの `other` ステムを解析

Demucsが未準備、またはWindows側の環境条件で動かない場合でも、解析サーバーは停止せずに簡易パート抽出へ切り替わります。

- ベース: 低域フィルターで抽出
- ギター: 中高域フィルターで抽出

通常はDemucs分離を優先し、失敗した時だけ簡易抽出を使います。

```powershell
.\scripts\run-server.ps1
```

成功すると、以下のような表示になります。

```text
MEFX analysis server listening on http://localhost:8787
```

確認URL:

```text
http://localhost:8787/health
```

## Step 3: Webアプリを起動する

別のPowerShellを開いて、以下を実行します。

```powershell
.\scripts\run-web.ps1
```

表示されたURLをブラウザで開きます。

通常は:

```text
http://localhost:5173
```

## Step 4: ESP32なしでシミュレーター確認する

Webアプリを開いたら、右上の `Simulator` ボタンを押します。

確認すること:

- `Simulator connected` と表示される
- Device Presetsに `SIM Clean` が表示される
- Manual Editorで値を変えられる
- `Send to MEFX-S3` を押すとDevice Presetsに追加される
- Status欄の入力/出力/マイク値が動く

詳しくは [シミュレーター手順](シミュレーター手順.md) を見てください。

## Step 5: ESP-IDFを入れる

ESP32に書き込むにはESP-IDFが必要です。

公式手順:

- https://docs.espressif.com/projects/esp-idf/en/latest/esp32/get-started/windows-setup.html

公式ドキュメントではWindows向けにEspressif Installation Managerを使う方法が案内されています。

詳しくは [ESP-IDF導入メモ](ESP-IDF導入.md) を見てください。

インストール後に確認:

```powershell
idf.py --version
```

## Step 6: KiCadを開く

KiCadは以下の場所にインストールされています。

```text
C:\Program Files\KiCad\10.0\bin\kicad.exe
```

KiCadを起動したら、以下を開きます。

```text
docs/kicad/mefx_s3.kicad_pro
```

## Step 7: ESP32ファームウェアをビルドする

ESP-IDFが入ったら以下を実行します。

```powershell
.\scripts\build-firmware.ps1
```

ESP32を接続して書き込むとき:

```powershell
.\scripts\flash-firmware.ps1
```

## Step 8: 部品を買う

最初からケースや基板発注まで進めず、まず机上テスト用の部品だけ買います。

見るファイル:

- [部品購入チェックリスト](checklists/procurement.md)
- [BOM](hardware/bom.md)

## Step 9: 電源から確認する

絶対に音声回路から始めないでください。

最初に確認するもの:

- 9V
- 5V
- 3.3V
- 4.5V仮想GND

見るファイル:

- [組み立て確認チェックリスト](checklists/bring-up.md)

## Step 10: 完成判定する

最後に以下をすべて満たしたらv1完成です。

- Webアプリでプリセットを作れる
- YouTube Tone Matchで仮プリセットを作れる
- ESP32へBluetooth送信できる
- ESP32が保存済みプリセットを保持する
- ギター/ベース入力で音が出る
- フットスイッチが動く
- 回路図、配線図、BOM、検証記録が揃っている

見るファイル:

- [完成判定チェックリスト](checklists/end-to-end.md)
