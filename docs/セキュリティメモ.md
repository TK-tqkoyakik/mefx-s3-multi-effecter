# セキュリティメモ

MEFX-S3試作 v1 を安全に開発・使用するための確認メモです。

## 今回対策済み

- ローカル解析サーバーのCORSを `*` から許可Origin方式に変更。
- 許可Origin以外から `/analyze` `/heartbeat` `/client-close` を呼べないように変更。
- 解析サーバーのJSONリクエストサイズを制限。
- 重いYouTube解析を同時に複数実行できないように変更。
- YouTube URLをURLパーサーで検証し、HTTPSのYouTube系ホストだけ許可。
- yt-dlpの取得にサイズ上限、動画時間上限、タイムアウト、リトライ上限を追加。
- Windowsセキュリティ対策として、Webページから `mefx://start` でPowerShellを自動起動する標準導線を削除。
- 解析サーバーは `scripts/start-local-analyzer.cmd` で手動起動する方式に変更。
- 初回の解析ツール導入は、手動で実行する方式に変更。
- npm依存は `package-lock.json` がある場合 `npm ci` を使うように変更。

## 主なリスクと考え方

### 1. ローカル解析サーバー

PC上でYouTube取得、音源分離、特徴量解析を行うため、最も注意が必要です。

- 公開Webから `127.0.0.1:8787` にアクセスします。
- 許可Origin以外は拒否します。
- 開発用にOriginを追加する場合は、信頼できるURLだけを `MEFX_ALLOWED_ORIGINS` に入れます。
- `HOST=0.0.0.0` で起動するとLAN内から触れる可能性があるため、通常は使いません。

### 2. ローカル起動

任意のWebサイトからPC内のPowerShellを自動起動する方式は、Windowsセキュリティに怪しい挙動として見られやすいため標準では使いません。

- 解析サーバーは `scripts/start-local-analyzer.cmd` を自分で起動します。
- 古い `mefx://` 登録が残っている場合は、`HKCU:\Software\Classes\mefx` を削除します。
- 知らないWebサイトから外部アプリ起動の確認が出た場合は許可しないでください。

### 3. 解析ツールのダウンロード

`yt-dlp`、Demucs、torch系パッケージはネットワークから取得します。

- 初回は時間がかかります。
- 無言インストールはしません。
- 将来的にはGitHub Releasesでバージョン固定済みランチャーを配布すると、さらに安全になります。

### 4. Bluetooth

Web Bluetoothはユーザーがデバイス選択画面でMEFX-S3を選んだ時だけ接続できます。

- 現状のESP32 GATTは試作用で、ペアリングやPIN認証は未実装です。
- 近くにいる第三者が別端末から接続できる可能性があります。
- 実機運用前には、ボタン長押し中だけペアリング可能にする、接続中は広告を止める、保存操作に簡易PINを要求する、などを検討します。

### 5. YouTube/著作権/利用規約

解析は一時取得した音声をPC内で処理する設計です。

- 音源は恒久保存しません。
- 再配布しません。
- 利用する動画や地域の規約に注意します。

### 6. ハードウェア

9V電源、アナログ音声、USB接続が関わります。

- 極性、GND、仮想GND、3.3V/5Vの確認前にESP32やADC/DACを接続しないでください。
- 初回通電は電流制限できる電源か、少なくともテスターで段階確認します。
- PCやアンプへ接続する前に、出力DCオフセットを確認します。

## 今後の強化候補

- ローカル解析サーバーにワンタイムセッションキーを追加する。
- GitHub ReleasesでWindowsランチャーを署名付き配布する。
- BLEにペアリング/ボタン承認/PINを追加する。
- ESP32側にプリセットJSONのさらに厳密な長さ・文字種チェックを追加する。
- GitHub Dependabotを有効化して依存更新を検知する。
